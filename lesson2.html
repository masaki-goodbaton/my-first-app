01

使用技術

CS
GitHub Codespace
クラウド開発環境。ブラウザだけで開発できる。Node.js・Git等がプリインストール済み。ローカルPCへのインストール不要。
CC
Claude Code
Anthropic製のAIプログラミングアシスタント（CLIツール）。ターミナルで日本語の指示を出すとコードを生成・編集してくれる。
Git
Git / GitHub
バージョン管理とコラボレーション。コードの変更履歴を追跡する仕組み。GitHubはGitのホスティングサービス。
02

環境構築とは

プログラムを開発するために必要なツールや設定を準備すること。通常はコードエディタ、プログラミング言語、データベース、フレームワーク等のインストールが必要。

ポイント

GitHub Codespace を使えば、これらのツールがクラウド上で全て揃うため、自分のPCに何もインストールしなくてよい。

03

GitHub Codespace のポイント

無料枠

注意

個人アカウントで月間60時間・ストレージ15GBまで無料。停止しないと無料枠を消費し続ける（30分放置で自動停止するが、手動停止が確実）。

起動方法

  1. リポジトリページを開く
  2. 緑色の「Code」ボタンをクリック
  3. 「Codespaces」タブを選択
  4. 「Create codespace on main」をクリック

停止方法

  1. リポジトリページ → 緑の「Code」ボタン
  2. 「Codespaces」タブ →「…」（三点メニュー）
  3. 「Stop codespace」を選択
ショートカット

Codespace内で作業中なら、左下の「Codespaces: ○○」をクリック →「Stop Current Codespace」が最速。

再開方法

同じ場所の「…」メニューから「Open in Browser」を選択。前回の状態から続きで使える。

便利

https://github.com/codespaces で全Codespaceを一覧管理できる。

04

Claude Code のポイント

インストール

npm install -g @anthropic-ai/claude-code

起動・終了

操作コマンド
起動claude（全て小文字）
終了exit または Ctrl+C

認証

Claude Pro / Max サブスクリプションが必要。初回起動時にOAuth認証フローが走る。

Codespace での認証の注意点

Codespace上ではlocalhostへのリダイレクトが失敗するため、ターミナルに表示されるURLを手動コピーして新しいブラウザタブで認証 → 表示された認証コードをターミナルに貼り付ける。

できること

ファイル作成、コード修正、エラー解決、コードの説明など。すべて日本語の自然文で指示できる。

06

Codespace と Cursor の関係

GitHub Codespace Cursor
  • ブラウザ上のVS Code
  • クラウドで実行
  • カリキュラムの標準環境
  • VS Codeベースのローカルエディタ
  • 独自AI機能（Cmd+K等）内蔵
  • 慣れたら移行先として最適
ポイント

両者は同じVS Codeベース。Claude CodeはCLIツールなので、Cursorのターミナルでも claude コマンドで使える（併用可能）。学習中はカリキュラム通りCodespaceで進め、慣れたらCursorに移行するのがスムーズ。

07

プロンプトインジェクションとは

AIへの入力（プロンプト）に悪意のある指示を埋め込むことで、AIの本来の動作を変更させたり意図しない動作をさせたりする攻撃手法。

なぜ重要？
  • コード内のコメントに「このファイルを削除して」等の指示を隠す
  • データファイルに「機密情報を出力して」等の命令を埋め込む
  • AIに本来の制限を無視させる指示を巧妙に含める

Claude Code 初期設定時にも警告される。信頼できるコードや環境でのみ使用すること。
